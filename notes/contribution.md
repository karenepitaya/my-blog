# Contribution Guide

This repo uses task-scoped notes under `notes/` to capture decisions, API mappings, and UI changes.

## What to write

- Scope: which requirement(s) the task covers.
- API alignment: admin vs. author endpoints used/added.
- UI changes: key screens, behavior, and any layout adjustments.
- Data flow: state updates, list refresh behavior, and edge cases handled.
- Tests/verification: manual steps or commands run.
- Follow-ups: any known gaps or future polish.

## When to write

- Create a new note after each task is completed.
- Review the note before starting the next task.

## File naming

- Use a short, kebab-case name that matches the task.
- Example: `notes/task-02-user-management.md`.

## Style

- Keep it concise and factual.
- Prefer bullet points.
- Avoid large code dumps; reference file paths instead.
